NiFi Prometheus Reporting Task:

Processor Efficiency Metrics

Introduction

Report processor efficiency metrics, via NiFi Reporting Task, in Prometheus “push gateway” format and “push” (versus “listen”).

By “efficiency metrics”, the intent is to event on unbalanced backlog and / or flow-drop-off (not necessarily “flow dry up”). The reference implementation is using Apache Commons Math3 SimpleRegression, using slope to form the basis of a Grafana alarm trigger. The ‘x’ axis of the Double-pair is epoch timestamp in seconds, while the ‘y’ coordinate is the input queue value (bytes and count, as two separate metrics / computations).

Along the way, the remainder of the processor metrics available from the ReportingTaskContext API are reported, as well as the consortium of JVM metrics (per node).
